---
name: commit
description: Commit the pending changes in the current branch. Use when a users wants to commit their changes.
---

Create a complete but concise commit message describing the pending changes in the current branch. Then commit the changes.

Commit messages should always include a body paragraph, not just a summary title. 
Focus on the "why" and key design decisions rather than listing files.
Ignore the brevity of existing commits in the log.
